AI Ops SOP Pack is a public Markdown documentation pack.

It is published as a canonical GitHub repository under CC BY 4.0.

## What is public here

- SOPs for AI-assisted engineering operations.
- Templates for handoff, PR audit, OOM recovery, and cold start.
- A fictitious filled example.
- Release notes and changelog.
- Source mapping and provenance notes.
- Documentation around public visibility and boundaries.

## What is usable now

The pack can be used as reference material for:

- structuring agent/operator handoffs;
- reducing false readiness claims;
- documenting PR final audit checks;
- recovering from context loss after editor or workstation crashes;
- separating evidence, review, approval, and permission-to-act.

## What must be adapted

Any team using these SOPs must replace:

- repository names;
- branch names;
- local commands;
- CI systems;
- validation commands;
- permissions;
- approval gates;
- incident procedures;
- organization-specific terminology.

Examples are illustrative only.

## What this repo does not prove

This repository does not prove:

- production readiness;
- certification;
- safety in another environment;
- runtime enforcement;
- provider integration;
- commercial readiness;
- permission-to-act;
- permission-to-merge;
- permission-to-trade;
- permission-to-publish.
