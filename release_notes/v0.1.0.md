# Release Notes — v0.1.0

AI Ops SOP Pack v0.1.0 is a published documentation-pack usability polish after the initial public V0 publication.

It improves external readability, adaptation safety, and practical reuse while preserving the original boundaries.

## Release type

Documentation-only release.

## What changed

### External entry guide

Added `START_HERE.md` as the first reading surface for external users.

It clarifies:

- what the repository is and is not
- intended reader
- recommended reading order
- portable vs ASSO-derived files
- safe adaptation boundaries
- conservative decision vocabulary
- stop conditions
- current publication status

### Templates

Added copyable blank templates:

- `templates/agent_handoff_template.md`
- `templates/pr_final_audit_checklist.md`
- `templates/oom_recovery_checklist.md`
- `templates/cold_start_checklist.md`

Templates are evidence-capture surfaces only. They do not grant approval, merge authority, runtime authority, or permission-to-act.

### Filled example

Added:

- `examples/pr_crash_recovery_filled_example.md`

The example is fictitious and non-live. It shows how OOM recovery, PR final audit, and agent handoff can be combined conservatively.

The example does not transfer `PASS`, readiness, approval, runtime authority, or permission-to-act to another environment.

### Status alignment

Aligned residual source/DRAFT wording in public SOP files with the current public repository status.

Historical source closeout status is retained as provenance only.

### Case-study classification

Clarified that `content/05_operator_guide.md` is an ASSO-derived local text-only case study, not a generic drop-in SOP.

## What did not change

v0.1.0 does not add:

- runtime capability
- provider integration
- live execution
- background daemon / watcher
- memory or ledger writes
- trading, order routing, leverage, wallet, or portfolio authority
- product offer
- commercial channel
- PDF distribution
- external announcement
- permission-to-act

## Recommended use

Start with:

1. `README.md`
2. `START_HERE.md`
3. `content/04_agent_handoff_format.md`
4. matching template from `templates/`
5. `examples/pr_crash_recovery_filled_example.md` if a filled walkthrough is needed

## Boundary statement

AI Ops SOP Pack remains a public Markdown documentation pack under CC BY 4.0.

It is not a runtime, product, service, certification, or operational authority.
